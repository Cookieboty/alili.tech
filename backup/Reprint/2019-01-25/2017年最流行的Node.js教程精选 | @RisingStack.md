            <h1>RisingStack 2017年最流行的Node.js教程精选</h1>
<p>Mark Hinkle，Node基金会执行总监，曾说道：‘对于Node.js，2017年是杰出的一年’。这点我们完全同意。 Node.js在技术领域有着令人羡慕的地位，每天线上运行的Node实例超过8.8百万个，并且这个数字在过去一年里飞涨。同时在一年时间内，Node.js的贡献者已经从1,100人发展到1,500人，每周下载的npm包超过30亿个。</p>
<p><strong>Node在蓬勃发展</strong></p>
<p>2017年对于RisingStack也是重要的一年。这一年，我们提供了许多的学习机会，不仅为<a href="">初学者</a>，也为<a href="https://blog.risingstack.com/tag/advanced-node-js-tutorials/">高级Node开发者</a>提供了很多的新教程，并且涵盖了微服务和Node的新特性。我们很高兴地看到，2017年有超过150万用户从我们这里学到了一些东西。</p>
<p><img src="http://p0.qhimg.com/t01d9aa6759c0c780f3.png" alt="Screen-Shot-2017-12-21-at-1.54.51-PM-1"></p>
<p>这也意味着通过RisingStack博客学习Node.js的开发者几乎比过去一年翻了一倍。 </p>
<p>在2018年，我们有许多新的想法要写：优秀的教程、深度学习案例、Node新特性以及更多。<strong>然而，为了迎接即将到来的挑战，并告别2017年，我们精选了今年最好的文章</strong></p>
<p>我们通过分析浏览量、分享数以及查询结果来确定2017年我们创建的最好的文章。他们就是：</p>
<h3>1. <a href="https://blog.risingstack.com/10-best-practices-for-writing-node-js-rest-apis/">编写Node.js REST APIs的10个最佳实践</a></h3>
<p>我们第一篇2017年度精选文章带你领略编写Node.js REST APIs的十个最佳实践。你可以学习如何正确使用HTTP方法，API路由和HTTP状态码。我们帮你测试你的Node.js REST APIs，并为Node.js REST APIs选择合适的框架。</p>
<p>鉴权功能、请求执行差异化、请求速限制和创建API文档，都是写好Node.js REST APIs的重要因素。这篇文章里都有介绍到。</p>
<h3>2.<a href="https://blog.risingstack.com/node-js-interview-questions-and-answers-2017/">Node.js面试问题及答案（2017版本）</a></h3>
<p>你准备做Node.js开发的面试官？还是打算去做面试者？不管如何，我们这里都有好消息给你。</p>
<p>我们结合2017年最新的内容，更新上个版本的Node的问与答。该新版本包含promise、Yarn、如何保护HTTP cookie免受XSS攻击以及确保依赖安全等部分。</p>
<p>在这个全面的问答集合帮助下，你可以快速的回顾Node里面最重要的主题。</p>
<h3>3. <a href="https://blog.risingstack.com/designing-microservices-architecture-for-failure/">为错误设计微服务架构</a></h3>
<p>通过过去四年在RisingStack Node.js咨询与开发业务里面获取的经验，我们收集了最为常见的技术与架构模式，并推荐你去使用，如果你想要构建和操作一个可用的高质量微服务。</p>
<p>微服务系统需要多个物理逻辑组件间的合作，若通过网络联系沟通而不是内存调用，将导致微服务系统额外的延迟和复杂度，而这种合作联系是非常重要的。在这篇文章中，我们讨论了实施微服务模式的风险，以帮助你检测、隔离和解决问题。</p>
<h3>4.<a href="https://blog.risingstack.com/mastering-async-await-in-nodejs/">精通Node.js的Async/Await</a></h3>
<p>异步结构在其他语言中已经有一段时间了。随着Node.js 8的发布，期待已久的async函数在Node.js上线了。</p>
<p>你可以在本文中学习，如何通过Node.js的async/await函数来简化回调和Promise。</p>
<p>假如你喜欢用<code>if-else</code>条件和<code>for/while</code>循环这种好语句，或者认为<code>try-catch</code>语句是错误处理的方式，那么你可愉快的用<code>async/await</code>来重写代码。</p>
<h3>5. <a href="https://blog.risingstack.com/node-js-mysql-example-handling-hundred-gigabytes-of-data/">Node.js + MySQL示例:处理上百GB的数据</a></h3>
<p>在这个Node.js和MySQL的例子项目中，你可以学到如何高效的处理数十亿行数据，而这些数据将占用上百GB的存储空间。通过例子，来看分区表，以及如何从数据库中增删数据。</p>
<p>这篇文章验证了，符合ACID原则的数据库管理系统方案像MySQL，也可以处理海量数据。所以你不必放弃事务型数据库的特色功能。</p>
<h3>6. <a href="https://blog.risingstack.com/yarn-vs-npm-node-js-package-managers/">Yarn vs npm - Node.js包管理器的现状</a></h3>
<p>Npm 4成为了Node 7.4默认的包管理器，但是与此同时，Facebook也上线了他的包管理器-Yarn。</p>
<p>让我们看看Node.js包管理器的现状，当要选择的时候，看他们都能为你做什么。</p>
<h3>7. <a href="https://blog.risingstack.com/node-js-async-best-practices-avoiding-callback-hell-node-js-at-scale/">Node.js异步最佳实践以及避免回调地狱</a></h3>
<p>一定要避免回调地狱！我们找了最好的技术工具来处理异步操作：async.js、promises、基于generator的流控制和异步函数。</p>
<p>坚持以下原则来让它变的更顺利：</p>
<ul>
<li><p>选择异步而非同步API，因为非阻塞语句比同步的要有更好的性能。</p>
</li>
<li><p>始终用最合适的流程控制或者其组合，来减少等待I/O完成的时间。</p>
</li>
</ul>
<h3>8. <a href="https://blog.risingstack.com/important-features-fixes-node-js-version-8/">Node.js 8重要的特性和修复</a></h3>
<p>期待已久的Node 8在5月30号发布了，带来许多新特性和改进。这篇文章中，我们要介绍最重要的部分，回顾一下异步钩子API、N-API和Node 8其他显著的变化。</p>
<p>并且，在今年的博客中我们也多次介绍到了这些。</p>
<p>Node.js 8.4.0发布的时候，我们<a href="https://blog.risingstack.com/node-js-http-2-push/">介绍HTTP/2服务推送最重要的部分</a>，同时介绍如何创建一个使用它的小型Node.js应用。</p>
<p>后来因为Node.js的新特性很棒，我们写了另外一篇<a href="https://blog.risingstack.com/whats-new-in-node-js-8-5/">处理ES模块，性能钩子和文件模块</a>文章 。</p>
<h3>9. <a href="https://blog.risingstack.com/building-an-api-gateway-using-nodejs/">用Node.js搭建API网关</a></h3>
<p>什么是一个API网关？为什么我们要用到它？这篇文章里，我们讨论了路由和版本管理、身份认证、数据合并、序列化、协议转换。同时试着去写一个简单的API网关。</p>
<p>我们将讨论为API网关定义明确的职责的重要性，以及仅包含通用共享逻辑的重要性。API网关有助于让你的服务小而专，所以正确合理使用是很重要的。</p>
<h3>10. <a href="https://blog.risingstack.com/building-a-node-js-app-with-typescript-tutorial/">用TypeScript构建Node.js教程</a></h3>
<p>这篇文章里，我们创建了一个简单的项目，来展示如何去构建、测试和调试一个用TypeScript写的Node.js应用。讨论了Typescript是什么，为什么使用它有益，以及如何用TypeScript开始项目。文章中也包括添加检查，写测试和调试部分。</p>
<h3>11. <a href="https://blog.risingstack.com/getting-node-js-testing-and-tdd-right-node-js-at-scale/">正确地使用Node.js测试和TDD</a></h3>
<p>想想你自己的未来！你想要在没有正确测试的代码上开发吗？如果不要，读读这篇关于正确地使用Node.js测试和TDD（测试驱动开发）的文章。测试驱动开发是一种先为模块写测试，再写实际开发的方法。</p>
<p>这篇指南包含了在RisingStack上我们所做的TDD的绝大部分内容：</p>
<ul>
<li><p>写错误的测试代码</p>
</li>
<li><p>写正确的测试代码</p>
</li>
<li><p>重构。</p>
</li>
</ul>
<h3>12. <a href="https://blog.risingstack.com/node-js-performance-monitoring-with-prometheus/">用Prometheus进行Node.js的性能监控</a></h3>
<p>在这篇文章中，讨论在生产环境中用Node.js应用时，具体要监控些什么。我们告诉你如何检测你的系统，选择监控工具时如何考虑，以及为什么我们推荐Prometheus当作一个自我托管的监控套件。</p>
<p>我们也有示例，给你更深入的意见，以便你选择它作为你的node.js应用的监控方式。</p>
<h3>13. <a href="https://blog.risingstack.com/node-js-native-modules-with-rust/">用Rust写快速和安全的原生Node.js模块</a></h3>
<p>当达到Node.js能提供的最高速率时，你可以做什么？为了提高你的应用性能，一个可能的方案就是用Rust而不是C++写原生模块。这是个好选择，因为其提供的安全和易用性。</p>
<p>在这篇文章中，我们将引导你完成一个现代、快速和安全的原生模块。</p>
<h3>14. <a href="https://blog.risingstack.com/consumer-driven-contract-testing-with-pact/">用Pact来进行消费者驱动的契约测试</a></h3>
<p>RisingStack里，我们喜欢用微服务，因为这种架构给予我们灵活性和快速性。这篇文章中我们展示了，在你的Nodejs微服务里面，如何使用Pack框架来执行消费者驱动的契约测试。</p>
<p>我们用Pact创建模拟服务器，来进行客户端单元测试。用Pact Broker收集，并与正在运行的服务进行验证，以保证消费者和提供者的一致性。</p>
<h3>15. <a href="https://blog.risingstack.com/writing-a-javascript-framework-client-side-routing/">编写一个Javascript框架-客户端路由</a></h3>
<p>在这篇文章，你可以看到Javascript的客户端路由。我们将讨论服务端路由和客户端路由的不同，以及为什么要区分对待。这篇博客是作者的<a href="https://blog.risingstack.com/tag/nx-writing-a-javascript-framework/">《写一个Javascript框架》</a>系列的最后部分。</p>
<p>这个系列文章是关于一个叫NX的开源客户端框架。该系列里，Bertalan介绍了他写框架的时候，遇到的不得不克服的主要困难。</p>
<h3>15 + 1. <a href="https://blog.risingstack.com/node-js-bootcamp/">我如何在RisingStack的Node.js训练营中生存</a></h3>
<p>作为一名有前景的有才华的初级程序员，Robert Czinege这个夏天参加了RisingStack的训练营，但是他仍需要学习我们是如何为客户构建优秀的Node.js应用。</p>
<p>在他的文章里，谈到了RisingStack的node.js训练营，帮助他跟上队里的其他人。顺便说一下，这个训练营开放给每个学习Node.js的人，因为我们在GitHub上面开源了。所以完美一击冒牌者综合症并开始快速学习Node.js吧！</p>
