            <h1><a href="#协同编辑器的历史性清单"></a>协同编辑器的历史性清单</h1>
<p>按时间顺序快速列出主要协同编辑器的演变。</p>
<p>正如任何这样的清单一样，它必定会在一开始便提到被誉为“<a href="https://en.wikipedia.org/wiki/The_Mother_of_All_Demos">所有演示之母</a>the mother of all demos”，在这个演示里<a href="https://en.wikipedia.org/wiki/Douglas_Engelbart">道格·恩格尔巴特</a>Doug Engelbart早在 1968 年就描述了几乎所有软件的详尽清单。这不仅包括协同编辑器，还包括图形、编程和数学编辑器。</p>
<p>一切都始于那个演示，只不过软件的实现跟不上硬件的发展罢了。</p>
<blockquote>
<p>软件发展的速度比硬件提升的速度慢。——沃斯定律</p>
</blockquote>
<p>闲话少说，这里是我找到的可圈可点的协同编辑器的清单。我说“可圈可点”的意思是它们具有可圈可点的特征或实现细节。</p>
<p>项目</p>
<p>日期</p>
<p>平台</p>
<p>说明</p>
<p><a href="https://www.codingmonkeys.de/subethaedit/">SubEthaEdit</a></p>
<p>2003-2015？</p>
<p>仅 Mac</p>
<p>我能找到的首个协同的、实时的、多光标的编辑器， <a href="https://www.emacswiki.org/emacs/SubEthaEmacs">有个在 Emacs 上的逆向工程的尝试</a>却没有什么结果。</p>
<p><a href="http://docsynch.sourceforge.net/">DocSynch</a></p>
<p>2004-2007</p>
<p>？</p>
<p>建立于 IRC 之上！</p>
<p><a href="https://gobby.github.io/">Gobby</a></p>
<p>2005 至今</p>
<p>C，多平台</p>
<p>首个开源、稳固可靠的实现。 仍然存在！众所周知 <a href="http://infinote.0x539.de/libinfinity/API/libinfinity/">libinfinoted</a> 协议很难移植到其他编辑器中（例如： <a href="https://www.emacswiki.org/emacs/Rudel">Rudel</a> 不能在 Emacs 上实现此协议）。 2017 年 1 月发行的 0.7 版本添加了也许可以改善这种状况的 Python 绑定。 值得注意的插件: 自动保存到磁盘。</p>
<p><a href="https://ethercalc.net/">Ethercalc</a></p>
<p>2005 至今</p>
<p>Web，JavaScript</p>
<p>首个电子表格，随同 <a href="https://en.wikipedia.org/wiki/Google_docs">Google Docs</a>。</p>
<p><a href="https://web.archive.org/web/20060423192346/http://www.moonedit.com:80/">moonedit</a></p>
<p>2005-2008？</p>
<p>？</p>
<p>原网站已关闭。其他用户的光标可见并且会模仿击键的声音。 包括一个计算器和音乐定序器。</p>
<p><a href="http://www.synchroedit.com/">synchroedit</a></p>
<p>2006-2007</p>
<p>？</p>
<p>首个 Web 应用。</p>
<p><a href="http://wiki.inkscape.org/wiki/index.php/WhiteBoard">Inkscape</a></p>
<p>2007-2011</p>
<p>C++</p>
<p>首个具备协同功能的图形编辑器，其背后的“whiteboard” 插件构建于 Jabber 之上，现已停摆。</p>
<p><a href="https://en.wikipedia.org/wiki/AbiWord">Abiword</a></p>
<p>2008 至今</p>
<p>C++</p>
<p>首个文字处理器。</p>
<p><a href="http://etherpad.org/">Etherpad</a></p>
<p>2008 至今</p>
<p>Web</p>
<p>首款稳定的 Web 应用。 最初在 2008 年被开发时是一款大型 Java 应用，在 2009 年被谷歌收购并开源，然后在 2011 年被用 Node.JS 重写。使用广泛。</p>
<p><a href="https://en.wikipedia.org/wiki/Apache_Wave">Wave</a></p>
<p>2009-2010</p>
<p>Web， Java</p>
<p>在大一统协议的尝试上失败。</p>
<p><a href="https://en.wikipedia.org/wiki/Conflict-free_replicated_data_type">CRDT</a></p>
<p>2011</p>
<p>特定平台</p>
<p>在不同电脑间可靠地复制一个文件的数据结构的标准。</p>
<p><a href="http://operational-transformation.github.io/">Operational transform</a></p>
<p>2013</p>
<p>特定平台</p>
<p>与 CRDT 类似，然而确切地说，两者是不同的。</p>
<p><a href="https://floobits.com/">Floobits</a></p>
<p>2013 至今</p>
<p>？</p>
<p>商业软件，但有对各种编辑器的开源插件。</p>
<p><a href="https://wiki.documentfoundation.org/Development/LibreOffice_Online">LibreOffice Online</a></p>
<p>2015至今</p>
<p>Web</p>
<p>免费的 Google docs 替代品，现已集成到 <a href="https://nextcloud.com/collaboraonline/">Nextcloud</a></p>
<p><a href="https://hackmd.io/">HackMD</a></p>
<p>2015 至今</p>
<p>？</p>
<p>商业软件，<a href="https://github.com/hackmdio/hackmd">开源</a>。灵感来自于（已被 Dropbox 收购的) hackpad。</p>
<p><a href="https://cryptpad.fr/">Cryptpad</a></p>
<p>2016 至今</p>
<p>Web ?</p>
<p>Xwiki 的副产品。服务器端的加密的、“零知识” 产品。</p>
<p><a href="https://prosemirror.net/">Prosemirror</a></p>
<p>2016 至今</p>
<p>Web， Node.JS</p>
<p>“试图架起消除 Markdown 文本编辑和传统的所见即所得编辑器之间隔阂的桥梁。”不是完全意义上的编辑器，而是一种可以用来构建编辑器的工具。</p>
<p><a href="https://quilljs.com/">Qill</a></p>
<p>2013 至今</p>
<p>Web， Node.JS</p>
<p>富文本编辑器，同时也是 JavaScript 编辑器。不确定是否是协同式的。</p>
<p><a href="https://teletype.atom.io/">Teletype</a></p>
<p>2017 至今</p>
<p>WebRTC， Node.JS</p>
<p>为 GitHub 的 <a href="https://atom.io">Atom 编辑器</a> 引入了“门户”的思路 ，使得访客可以夸多个文档跟踪主人的操作。访问介绍服务器后使用实时通讯的点对点技术（P2P），基于 CRDT。</p>
<p><a href="http://typeintandem.com/">Tandem</a></p>
<p>2018 至今</p>
<p>Node.JS？</p>
<p>Atom、 Vim、Neovim、 Sublime 等的插件。 使用中继来设置基于 CRDT 的 P2P 连接。多亏 Debian 开发者的参与，<a href="https://github.com/typeintandem/tandem/issues/131">可疑证书问题</a>已被解决，这使它成为很有希望在未来被遵循的标准。</p>
